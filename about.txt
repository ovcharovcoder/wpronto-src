═══════════════════════════════════════════════════════════════
                            DESCRIPTION
═══════════════════════════════════════════════════════════════

WPronto is a local WordPress development environment for Windows.
It provides a complete local development stack including Nginx, PHP, and MySQL,
allowing you to create and manage multiple WordPress sites with one click -
no manual configuration, no hosting required.

✅ One-click start/stop of full server stack (Nginx + PHP + MySQL)

═══════════════════════════════════════════════════════════════
                    HOW TO RESTORE FROM BACKUP
═══════════════════════════════════════════════════════════════

IMPORTANT: WPronto does not have an automatic one-click restore feature yet.
To restore a site from backup, please follow these manual steps:

STEP 1: CREATE A NEW SITE
   • Open WPronto
   • Click "Create Site" button
   • Enter the EXACT SAME site name as the original site
   • Example: if backed up site was "mynewsite", create "mynewsite"

STEP 2: COPY BACKUP FILES
   • Open Windows Explorer
   • Navigate to: WPronto\backups\your-site-name\YYYY-MM-DD_HH-MM-SS\
   • Copy the entire contents of the "files" folder
   • Paste and OVERWRITE files in: WPronto\www\your-site-name\

STEP 3: RESTORE THE DATABASE
   • Open phpMyAdmin: http://localhost/phpmyadmin
   • Select the database: your-site-name_db
   • Click the "Import" tab
   • Choose the SQL file from backup folder (your-site-name_db.sql)
   • Click "Go" to import

STEP 4: RESTART THE SERVER
   • In WPronto, click "Stop" (or press Ctrl+X)
   • Click "Start" (or press Ctrl+S)

✅ Your site is now restored with all files and data!

═══════════════════════════════════════════════════════════════
                         QUICK START GUIDE
═══════════════════════════════════════════════════════════════

1. START THE SERVER
   Click the "Start" button OR press Ctrl+S on your keyboard.
   Wait for the status to change to "● SERVER RUNNING" (green).

2. CREATE A NEW SITE
   Click "Create Site" button.
   Enter a site name (letters, numbers, and hyphens only).
   Example: "mynewsite" → will be accessible at http://mynewsite.wp

3. INSTALL WORDPRESS
   After creation, click "Open Admin" button or open:
   http://your-site.wp/wp-admin/install.php
   Fill in your site details and complete the installation.

4. BACKUP YOUR SITE
   Select a site from the list → Click "Backup Site" (green button)
   A full copy of files and database will be saved to:
   WPronto\backups\your-site-name\YYYY-MM-DD_HH-MM-SS\

5. START DEVELOPING
   Access your site admin at: http://your-site.wp/wp-admin
   Upload plugins & themes up to 256MB!
   Manage databases via phpMyAdmin: http://localhost/phpmyadmin

6. MANAGE SITES
   • Open Admin    : Select site → Click "Open Admin"
   • Backup Site   : Select site → Click "Backup Site" (green)
   • Delete Site   : Select site → Click "Delete Site" (red)
   • Stop Server   : Click "Stop" OR press Ctrl+X


═══════════════════════════════════════════════════════════════
                         FILE STRUCTURE
═══════════════════════════════════════════════════════════════

WPronto\
├── core\           (Nginx, PHP, MySQL, phpMyAdmin)
├── config\         (Nginx & PHP configuration files)
├── data\           (MySQL databases storage)
├── template\       (WordPress template for new sites)
├── www\            (Your WordPress sites folders)
│   ├── default\    (Default localhost site)
│   └── your-site\  (Your created sites)
├── backups\        (Site backups: files + database exports)
│   └── site-name\  (Backups organized by site)
│       └── YYYY-MM-DD_HH-MM-SS\ (Timestamped backup folders)
│           ├── files\           (Complete site files)
│           └── site-name_db.sql (Database dump)
├── logs\           (Nginx, PHP, MySQL, WPLaunch error logs)
└── tmp\            (Temporary files)


═══════════════════════════════════════════════════════════════
                         TROUBLESHOOTING
═══════════════════════════════════════════════════════════════

❌ Site shows 404 Not Found
   → Restart the server (Stop → Start or Ctrl+X → Ctrl+S)
   → Check if files were copied to www/your-site/

❌ Can't start server (port 80 in use)
   → WPronto automatically switches to port 8080
   → Check the log for "using alternative ports" message
   → Or manually stop Skype, Teams, or other web servers

❌ Database connection error during WordPress install
   → Database name: your-site_db
   → Username: root
   → Password: (leave empty)
   → Host: 127.0.0.1

❌ Restored site not showing in list
   → Make sure you created a new site with the EXACT same name
   → Verify that files were copied to www\your-site-name\
   → Restart the server after restoring

❌ Restored site shows database connection error
   → Make sure you imported the SQL file via phpMyAdmin
   → Verify the database name matches your site name + "_db"
   → Check that username is "root" with no password
